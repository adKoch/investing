# Future Tasks & Plans

This directory holds future plans, checklists, proposed strategies, and research tasks that we plan to address.

## Workflow

1. **Create a Task**: When we identify a future plan or strategy we want to design, we create a new markdown file in `/tasks/` (e.g. `/tasks/backtest-trend-following.md`).
2. **Execute & Track**: The file will serve as a checklist. We update it as we make progress.
3. **Move to Done**: Once the plan is fully executed, verified, and integrated into our main code/documentation, the markdown file is moved into the `/tasks/done/` subdirectory. This keeps the active `/tasks/` folder uncluttered.

---

## Recommended Task File Template

When creating a new task, you can copy the following template:

```markdown
# Task: [Task Title]

- **Status**: [Planned / In Progress]
- **Target Date**: [YYYY-MM-DD or TBD]
- **Description**: Brief explanation of what this task intends to accomplish.

## Checklist

- [ ] Item 1
- [ ] Item 2

## Notes & Resources
- Useful links, algorithms, or formulas.
```
